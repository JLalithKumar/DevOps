Exercise: Node NotReady Production Incident

Status: Completed Successfully

What was performed: - Verified node health using kubectl get nodes and

Checked filesystem usage with df -h. - Simulated disk exhaustion by
creating large files using fallocate. - Observed kubelet events: -
FreeDiskSpaceFailed - ImageGCFailed - EvictionThresholdMet - Verified
DiskPressure=True. - Observed the automatic
node.kubernetes.io/disk-pressure:NoSchedule taint. - Removed the test
files and restarted the kubelet. - Confirmed recovery with: -
DiskPressure=False - Ready=True - Taints removed

Outcome: Successfully reproduced and recovered from a Kubernetes node
disk pressure incident on Amazon EKS.
